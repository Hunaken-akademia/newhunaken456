# レース結果「着順0艇」根本修正 v117

## ログから確定した原因

`ST6艇・着順0艇` が全場で続いたため、結果未確定ではなく着順パーサーの共通障害でした。
旧版は、公式結果ページ内の表ヘッダーに「着」「枠」「ボートレーサー/選手名」がすべて含まれる表だけを解析していました。
公式側の表記・HTML差分でこの条件を満たさず、着順表を丸ごと除外していました。
STだけ6艇取れていたのは、別のスタート情報フォールバックが動いていたためです。

## 修正内容

- 結果表ヘッダーの厳しすぎる条件を廃止
- 1着〜6着の行を行単位で判定
- 全角数字（１着など）に対応
- 艇色class、登録番号、艇番セルの三重照合
- table抽出失敗時は全trを再解析
- 最後にテキスト化した着順ブロックを登録番号で照合
- 公式サイト向けReferer/User-Agentへ分離
- 外部取得に12〜15秒タイムアウトを追加
- Actionsの結果回収を4並列化
- 保存0件なのに緑完了しないよう、0件時は失敗終了
- PENDINGログにHTML長・結果文字有無・parserVersionを追加

## 風

風は引き続き完全手動です。
API・事前取得・レース変更・オールクリアから風を自動上書きしません。

## SQL

今回の修正では追加SQLはありません。
`sql/enable_daily_result_capture.sql` を再実行する必要はありません。

## 反映後の確認

GitHub Actionsで日付 `2026-07-09`、dry `false` を実行してください。
正常ならログに次の形式が出ます。

`SAVED 桐生1R rows=6 ... parser=v117`

まだPENDINGの場合は、先頭1件の `html=... hasResultText=... parser=...` を確認してください。
