# 購入ルール固定 完全版

この版では、Stripe購入者自動追加のルールを以下に固定しました。

## 販売ルール

- 購入は1人1回まで
- 既に `paid_users` に同じGoogleメールがある場合は、Stripe決済ページを作成しない
- 販売期間は JST 基準で `2026/7/13 00:00` 〜 `2026/8/13 23:59`
- 利用期限は全員共通で JST `2026/12/31 23:59`
- 決済成功後の `expires_at` は常に `2026-12-31T14:59:00.000Z`
- 期限延長・再購入延長はしない

## 変更ファイル

- `api/create-checkout-session.js`
  - 販売期間チェック追加
  - 既存購入者チェック追加
  - 既存メールは409で購入不可
  - Stripe metadataに販売期間/利用期限を保存

- `api/stripe-webhook.js`
  - 6ヶ月延長ロジックを削除
  - 固定期限 `2026/12/31 23:59 JST` に変更
  - 既存メールがある場合は上書き/延長せず無視

- `sql/setup_paid_users_and_rls.sql`
  - plan初期値を `buyout_2026` に変更
  - 販売期間・利用期限のメモを追加
  - 手動テストSQLを固定期限に変更

- `src/App.jsx`
  - 購入画面の説明文を固定販売期間・固定利用期限に合わせて変更

## 注意

販売期間外は `/api/create-checkout-session` が403を返し、Stripe決済画面は作成されません。
Webhook側でも販売期間外の決済sessionは拒否します。

ただし、Stripe上で既に作成されたCheckout Sessionの有効期限などはStripe仕様に依存します。
運用上は販売終了後にStripe側の価格・リンク・販売導線も停止してください。
