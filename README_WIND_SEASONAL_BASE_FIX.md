# 風点差説明・場平均季節別DB優先対応

## 変更内容

- 場平均1着率の優先順位を変更
  1. DB季節別場平均（venue_base_season）
  2. DB通年場平均（venue_base）
  3. 固定季節別場平均（VENUES_SEASONAL）
  4. 固定通年場平均（VENUES）

- correction_tables の venue_base_season をアプリ側で読み取るように対応。

## 注意

同じレースでも、手入力版と自動取得版で点数がズレる主因は以下。

- 自動取得版は風が入る
- 手入力版は風が未入力/無風扱いになりやすい
- 自動取得版はDB補正、コース別平均ST、DB成績、展示保存系を使う
- 手入力版は古い固定ロジックに近い

今後は自動取得版を正とする。
