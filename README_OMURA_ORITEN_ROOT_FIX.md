# 大村10R 展示未取得の根本対策

## 原因
BOATCASTのオリジナル展示(oriten)系テキストは、場・レースによって列形式が揺れる。
従来は `艇番\t選手名\t一周\tまわり足\t直線` 形式を主に想定していたが、
大村10Rのように `登録番号\t選手名\t一周\tまわり足\t直線` または艇番なしの行順形式になると、
一周・まわり足・直線が画面上に出ていてもアプリ側では6艇分として結合できず、
「展示情報不足」と判定される可能性があった。

## 修正
`parseBoatcastOritenRows` を強化し、以下を全場共通で処理するようにした。

- 艇番つき: `艇番 / 選手名 / 一周 / まわり足 / 直線`
- 登録番号つき: `登録番号 / 選手名 / 一周 / まわり足 / 直線`
- 艇番なし: 行順を1〜6号艇として採用
- 登録番号・年齢など余計な数字が混ざっても、タイム範囲から `一周→まわり足→直線` の連番を探索

これにより、大村に限らず、BOATCAST側の列揺れで展示が未取得になるケースを減らす。

## 注意
展示そのものがBOATCASTで未公開の場合は、引き続き「公開展示待ち」または「展示情報不足」になる。
