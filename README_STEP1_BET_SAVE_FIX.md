# STEP 1：買い目・予想履歴の保存修正

## 修正対象

- `src/App.jsx`
- `dist/assets/index-v121-bet-save-fix-v1.js`
- `dist/cloud-bootstrap-v3.js`
- `dist/index.html`

## 修正内容

1. `setState` の updater 内で外側変数へ代入し、その直後に保存していた処理を廃止。
2. `betRecordsRef` / `recordsRef` で最新配列を同期保持。
3. `persistBets` / `persistRecords` は state 更新前に同期的に次の配列を計算してから保存。
4. `undefined` または配列以外の値は保存しない防御を追加。
5. 過去に `localStorage` / `window.storage` へ文字列 `"undefined"` が保存されていた場合、空配列として安全に復旧。
6. 買い目の保存先が両方とも失敗した場合は成功表示を出さず、カートを残して再試行できるように変更。
7. 本番配信用 `dist` も更新し、キャッシュ回避のため新しいアセット名へ変更。

## 変更していないもの

- 展示評価の数式・重み（展示＋1周＋回り足）
- レース結果・展示の自動取得
- `paid_users`
- Googleログイン
- note購入申請・承認
- Stripe決済・Webhook
- Supabaseテーブル・RLS・既存データ
- cloud-save-v3 のクラウド正本・楽観ロック
- 平均STの初期値「直近6ヶ月」

## 確認済み

- 配信用JavaScript：`node --check` 合格
- 括弧バランス：`{}` `()` `[]` すべて差分0
- 連続2回の保存で前の履歴を失わないこと
- `undefined` を保存しないこと
- 不正な非配列値では既存履歴を上書きしないこと
- 平均STの初期値が「直近6ヶ月」のままであること

## 反映後のテスト

1. 有料版を開く。
2. テスト用の買い目を1件記録する。
3. 画面下の「舟券の履歴」と収支へ即時反映されることを確認する。
4. クラウド欄に「未保存の変更があります」と出たら「今すぐ保存」を押す。
5. 保存完了後に再読み込みし、履歴が残ることを確認する。

現在のクラウド保存は手動方式です。ログイン中はクラウドが唯一の正本なので、再読み込み前に「今すぐ保存」を押してください。
