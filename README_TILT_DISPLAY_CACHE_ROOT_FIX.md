# TILT / DISPLAY / CACHE ROOT FIX

## 修正内容

### 1. チルト誤反映の根本対策
- BOATCASTの直前情報で、調整体重 `1.0` / `1.5` などをチルトとして誤認しないように修正。
- 自動取得では、符号付きの `+0.0` / `-0.5` / `+0.5` のようにチルトとして判別できる値だけを採用。
- 符号なしの `1.0` 以上は調整体重の可能性が高いため、チルト補正・伸び警戒には使わない。

### 2. 展示取得失敗の追加対策
- BOATCASTの `tkz / oriten / stt` txt の結合に失敗した場合、公式 `beforeinfo` を最終フォールバックとして解析。
- 大村10RのようにBOATCAST画面上は展示が出ているが、txt列揺れで6艇結合できないレースを救済。
- 大村限定ではなく、全場共通で最終フォールバックを入れている。

### 3. 古い不具合キャッシュ対策
- yoso_cache / Vercelメモリキャッシュのキーを `v112` に更新。
- 過去に保存された「チルト誤反映」「展示未取得」の古いキャッシュを読まないようにした。
- full取得は古いstaleキャッシュを優先返却しない設定に変更。

## 反映後の注意
- GitHub上書き後、Vercel Redeploy時は `Use existing Build Cache` を外す。
- 反映直後に同じレースを再取得し、チルト欄と展示データが更新されるか確認する。
