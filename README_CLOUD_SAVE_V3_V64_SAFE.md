# v121 クラウド保存 v3（v64安全方式）

## 目的
無料版v64で確立していた「ログイン中はクラウドを唯一の正とする」方式へ合わせ、空データ上書き・別アカウント混入・別端末の新しい保存の消失を防ぎます。

## 変更内容
- React本体を表示する前に、Googleログイン中の本人クラウドデータを読み込む
- 読込完了後にだけReact本体を起動する
- クラウドの `records` と `bet_records` を端末側へ完全置換
- 端末データとクラウドデータを自動マージしない
- アカウントが変わった場合、前アカウントの端末データを先に消去
- クラウド読込に失敗した場合、保存ボタンを無効化
- クラウド読込前の空データ保存を禁止
- 保存直前にクラウドを再確認し、無料版・別端末で新しい更新があれば上書きを禁止
- クラウドに履歴がある状態で0件保存する場合は確認画面を表示
- ログアウト時に端末側の予想履歴・収支履歴を消去（クラウド側は無傷）
- 保存上限：予想履歴100件、舟券収支300件
