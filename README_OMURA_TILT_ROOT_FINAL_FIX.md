# 大村10R 展示取得・チルト誤反映 根本修正 FINAL

## 修正内容

### 1. チルト誤反映の再修正
BOATCASTの直前情報で、調整体重 `1.0` / `1.5` がチルトとして反映されるケースを追加で防止しました。

- チルトは `+0.0` / `-0.5` など、明示的なチルト列・符号付き値を優先
- 数値だけの `1.0` は調整体重の可能性が高いため、チルトとして推測しない
- フロント側のフォールバック解析でも、展示タイム周辺の `1.0` をチルト推測しない
- APIから空チルトが返った場合は、画面側でも空欄のままにする

### 2. 大村10Rなどで展示取得できない問題の根本対策
BOATCASTのtxt/画面テキストで、艇番列が無い場合に `A1` / `A2` の数字を艇番と誤認していました。

例：
- `A1 /4096 石橋...` の `1` を1号艇と誤認
- `A2 /3611 岩崎...` の `2` を2号艇と誤認

このせいで5号艇・6号艇などが2号艇扱いになり、6艇分がそろわず「展示情報不足」になることがありました。

今回の修正で、

- 先頭セルが単独の `1〜6` の時だけ艇番として扱う
- 艇番列が無い形式では行順で1〜6号艇に割り当てる
- tkz/oriten/sttの結合が崩れた場合、行順ベースで再結合する

ようにしました。
