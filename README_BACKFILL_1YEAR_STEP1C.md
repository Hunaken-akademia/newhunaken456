# 過去1年バックフィル STEP1C：場名・レース番号取り違え診断

## 目的
STEP1BでST抽出は成功しましたが、同じ場・同じRに重複行が出ており、場名またはレース番号の追跡がずれている可能性があります。

このSTEP1CではDBへ一切書き込まず、以下を追加でログ出力します。

- 場・レースヘッダー検出履歴
- 重複候補の周辺行
- 6艇揃わないレースの周辺行
- 候補行ごとの直前ヘッダー行番号

## 安全性
- 本番 `race_results` へ書き込みません
- stagingへも書き込みません
- `paid_users`、note申請、購入者情報には触れません
- K票ファイルを読み、ログへ診断結果を出すだけです

## 実行
Actions → `inspect-k-file-backfill` → Run workflow

- date: `2026-07-02`
- show_raw: `false`

ログの以下を送ってください。

- `=== K票バックフィル事前診断 v3 ===` から `races_not_6_rows_sample` まで
- `場・レースヘッダー検出履歴`
- `問題候補の周辺コンテキスト`
