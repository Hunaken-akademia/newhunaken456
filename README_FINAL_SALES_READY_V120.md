# 販売前最終対応 v120

## 反映済み

- 購入前画面に、予想履歴・収支記録が端末内保存であることを明記
- 機種変更、ブラウザデータ削除、プライベートブラウズ、端末故障で記録が消える可能性を明記
- 端末間同期、クラウドバックアップ、消失記録の復元に未対応であることを明記
- アプリ下部の「ご利用にあたって」に同じ注意事項を追加
- 公開診断API `/api/checkout-health` は本番で404を返すよう無効化
- no-install / prebuilt方式のため `dist/` は削除せず維持
- Stripe決済、Webhook、Googleログイン、paid_usersゲート、販売期間、利用期限、クーポン対応を維持
- 結果自動回収ワークフロー、風の完全手動設定を維持

## 販売ページ・特商法表記で別途必要

次の項目は事業者情報が必要なため、このZIPでは自動入力していません。

- 販売事業者名
- 住所・電話番号（請求があった場合の開示表記を使う場合は、その条件）
- 連絡先メールアドレス
- 販売価格と支払時期
- 利用期限
- 返金・キャンセル方針
- 動作環境
- サポート範囲
- 的中・利益を保証しない旨
- 外部データ取得元の商用利用・許諾整理

## 結果自動回収の確認基準

GitHub Actions `capture-race-results` の実行後に、ログで次を確認してください。

- `savedRaces > 0`
- `savedRows > 0`
- `failed = 0` または原因を把握した少数のみ

Supabaseでも日付別に `race_results` のレース数、艇数、ST、着順を確認してください。
