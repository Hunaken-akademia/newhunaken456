# 鳴門風向き根本修正 + グレード別枠別成績即時反映

## 1. 鳴門で追い風・向かい風が逆になった根本原因

以前の風向き解析では、公式 beforeinfo HTML にある一般クラス名 `weather1` の末尾 `1` を、風向きアイコン番号1と誤認する可能性がありました。
そのため、実際の風アイコン `is-windN` を読まず、場別反転を追加してもレースによって逆判定が残りました。

### 今回の対策

- `weather1` のような一般クラスを風向き番号として絶対に採用しない
- `is-windN`、`data-wind-direction`、明示的な風向き画像ファイル名だけを採用
- 水面気象ブロックに複数の「追い風/向かい風」語が混在する場合は、凡例・説明文と判断して文字判定を不採用
- 現在Rの候補を `明示文字 > 明示属性 > 正確な is-windN` の順で統合
- 現在Rの情報がある時は、前Rの風で上書きしない
- キャッシュを v115 に更新

## 2. SG/G1/G2/G3ごとの枠別成績

フロントの集計ロジック自体は既に SG / G1 / G2 / G3 に対応しています。
今すぐ反映できるかは、`public.races.grade` が過去分に入っているかで決まります。

既存の `race_results` と `races` が保存済みなら、次のSQLを一度実行することで、race_title/race_typeから過去分を即時バックフィルできます。

`sql/backfill_race_grades_now.sql`

実行後、選手成績の区分プルダウンで SG / G1 / G2 / G3 を選ぶと、そのグレードだけのコース別1着率・2連対率・3連対率が反映されます。

### 注意

過去レースに `race_title` も `grade` も入っていない行は、自動分類の材料がないため未分類のままです。その分だけは今後の自動取得または別途タイトルのバックフィルが必要です。
