# 過去1年バックフィル STEP1D：場名ヘッダー厳格化＋レースタイム無し行の採用

## 目的
STEP1Cで以下の2点が確認できました。

1. 場名検出が広すぎて、レース名の「福岡選抜」を場名として拾ってしまう
2. 5・6着などでレースタイムが「.  .」の行を除外していたため、1レース6艇分が揃わない

STEP1Dでは、DBへ一切書き込まず、K票の読み取りロジックだけを再診断します。

## 変更内容
- 場名は「ボートレース○○」ヘッダーだけで判定
- 「ボートレース唐　津」のような場名中の空白を詰めて判定
- 「福岡選抜」のようなレース名では場名を切り替えない
- レースタイムが「.  .」でも、ST・進入・艇番・登番が読める行は採用

## 安全性
- 本番 `race_results` へ書き込みません
- stagingへも書き込みません
- `paid_users`、note申請、購入者情報には触れません
- K票ファイルを読み、ログへ診断結果を出すだけです

## 実行
Actions → `inspect-k-file-backfill` → Run workflow

- date: `2026-07-02`
- show_raw: `false`

理想値の目安:

- rows_without_st=0
- invalid_course_rows=0
- races_with_6_rows が大幅増
- races_not_6_rows が大幅減、できれば0
